Overview

In angular if we want to get the reference of our template into the component file then we can choose @ViewChild or @ViewChildren

To see the result of @ViewChild and @ViewChildren we have to use ngAfterViewInit() method of Angular Lifecycle hook because components view will be initialized after calling ngAfterViewInit().

To use ngAfterViewInit() method, the class need to implement AfterViewInit interface as below

export class ViewChildComponent implements OnInit,AfterViewInit {{ '{' }} }
In Javascript, if we want to get id of an element from the HTML then we used DOM methods
For Example,
<h1 id="heading">Heading </h1>;
let output = document.getElemntById('heading')
Similarly In Angular,
<h1 #heading>Heading </h1>;
@ViewChild('heading') output : ElementRef
ngAfterViewInit(){{ '{' }}
this.output.nativeElement.style.color = 'blue';
}

View Children Demo

ViewChildren can refer to more than one element with same template reference.Hence it uses QueryList

For Example,
<h1 #fc>This is the first content </h1>
<h1 #fc>This is the first content </h1>
@ViewChildren('fc') content : QueryList <any>
ngAfterViewInit(){{ '{' }}
this.content.first.nativeElement.style.color = 'red';
}
